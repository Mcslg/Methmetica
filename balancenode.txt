為了讓操作感覺自然，我們可以定義一套「代數手勢語言」：

操作目標	交互手勢	數學效果	範例

1. 歷史步驟自動套用（Macro / 轉換管線）
你的想法： 改變輸入後自動套用之前做過的調整。 我的分析：

正面：這是一個殺手級功能。這意味著這個節點不再只是一個單次解題的白板，而是變成了一個**「自定義推導函數」**。用戶只要手動操作過一次（例如示範如何解一元二次方程式），這個節點就學會了這個解法，可以套用到任何後續傳入的方程式。
考量（反面）：如果新的方程式結構不同（例如原本是 ax+b=c，輸入變成了 sin(x)=1），過去的步驟可能會套用失敗（找不到對應的項）。這需要我們設計一個「容錯機制」，當套用失敗時，節點停在最後成功的那一步，並高亮提示用戶接手操作。
2. 除以變數的條件提醒與不等式翻轉
你的想法： 除以 x 時跳出提醒，不等式則提醒正負。 我的分析：

正面：這解決了傳統 CAS（電腦代數系統）通常會硬生生吐出條件或直接忽略的問題，賦予了教育意義與數學嚴謹性。
實作設計：當執行危險操作（除以變數、開偶次方根）時，可以彈出一個極簡的小視窗：假設 x ≠ 0。 對於不等式 > 或 <，如果除以 x，系統可以問：「x 的符號是？」若使用者選擇「不確定」，我們可以讓這個節點的輸出**「分岔」**（Branching），同時產生兩條線路：一條是 x > 0 的結果，一條是 x < 0 且符號翻轉的結果。不過這會讓節點流向變複雜，初期可以先限定使用者選擇一個假設繼續進行。
3. 圈選括號與移入係數
你的想法： 圈起來放入括號，把係數或 x 拖到括號前面。 我的分析：

正面：視覺直覺，取代了繁瑣的鍵盤游標移動。拖曳係數到括號前相當於執行分配律的逆向或組合。
實作設計： 實體的「畫圈圈（Lasso）」在網頁上較難做到完美辨識（尤其是多個數學項交錯的狀況）。更可靠的做法是：
框選 (Marquee) 或 Shift+點擊：選取多個連續的項，選取後上方浮現一個 

( )
 按鈕，點擊即包裝。
當括號形成後，拖動另一個 x 或係數到括號「上面」或「旁邊」，系統會自動將其吸附為括號的乘數。
5. 兩邊同時取 Log、同時平方、同乘等全局操作
要對方程式「兩側」同時進行操作，我們需要一個代表**「整個等式」**的交互熱點（Hotspot）。

設計方案建議：以「等號 (=)」作為全局操作的樞紐。

當游標 Hover 或點擊 = 時， = 會變成一個互動操作台。
方案 A (快捷選單)：點擊 =，彈出一個環形或橫向的小選單，包含 

( )²
, √, log, ln, e^, | | 等常用全局操作。點擊後，兩邊自動被該函數包覆。
方案 B (鍵盤指令)：選中 =，直接在鍵盤按下 l o g，兩邊就會自動變成 log(左) = log(右)。
方案 C (拖曳賦予)：這也是你原本的想法。我們在節點側邊保留一個或多個「空白磚塊」，使用者可以輸入 x^2 或 log()，然後將這顆磚塊拖曳並丟進 = 號之中。系統偵測到丟在等號上，就會將其應用於兩側。
